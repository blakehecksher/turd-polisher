# 2026-05-17 2338 — v1 Build

## What was done
- Single-file `index.html` written (~36KB, ~900 lines including inline CSS + JS module).
- Three.js r160 via ESM CDN (`https://unpkg.com/three@0.160.0/build/three.module.js`).
- Procedural everything: geometry (turd archetypes + lathed toilet), shaders (turd surface, water caustics, background gradient), audio (WebAudio synthesis for all SFX), textures (sparkle/droplet/confetti drawn on canvas).
- Custom turd shader paints from a 256×128 RGBA8 DataTexture; each channel = one polish stat. Raycaster + UV-space radial paint with V-axis wrap-around.
- Smoke-tested in Playwright (Chromium): title click → drop animation → polish gauges update across all 4 tools → result screen with grade + stats → next-round transition advances to Round 2.

## What worked
- Self-contained ESM via direct unpkg import: no install, no bundler, runs from any static server.
- DataTexture (UnsignedByteType, RGBAFormat) at 256×128 plenty for the visible polish granularity at this scale; updates are cheap.
- LatheGeometry profile produced an acceptable cartoon-style toilet quickly.
- TubeGeometry over CatmullRomCurve3 + post-build vertex displacement gave decent turd shapes with low effort.

## What didn't and why
- Initial camera target (cameraTo) was too close — turd filled frame. Fixed by pulling camera back (y=2.8, z=5.2) and scaling turd to 0.85.
- `e.target.closest` blew up in the pointerdown handler when synthetic events fired with non-Element targets. Hardened with a typeof guard.
- `lessDirty` modifier on round 1 made the FILTH gauge appear pre-filled at 40% on round start, which masked whether brush was working during testing. Removed.
- Initial tool strengths (1.3–1.8) were too small to fill gauges in 15s. Bumped to 6.5–8.0 plus larger radii.

## Decisions made
- No external assets — every visual + audio asset procedurally generated. (Per spec.)
- Single HTML file, ESM import of Three.js from unpkg. (Per spec.)
- 4 polish channels packed into one RGBA texture, painted in-place from CPU. Avoided a render-to-texture solution because the resolution is small enough for CPU writes to be cheap and the painting math is clearer.
- Cartoon outline via back-face dilation, not post-process edge detection.
- Difficulty curve: shrink timer + add aggressive archetypes/modifiers from round 2+.
